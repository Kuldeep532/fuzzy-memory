## Firebase Setup
- `libs.versions.toml`: `firebaseBom = "33.1.0"`, `googleServices = "4.4.2"`, plugin alias `google-services`.
- `app/build.gradle.kts`: `alias(libs.plugins.google.services)` in plugins block + BOM + auth/firestore/messaging/play-auth deps.
- `app/google-services.json`: **Placeholder** — must be replaced from Firebase Console. Package: `com.nexuswavetech.nexusplus`.
- `R.string.default_web_client_id` auto-generated by google-services plugin from JSON.

## Auth Flow
- `FirebaseAuthRepository` wraps `FirebaseAuth.getInstance()` in try/catch; returns `AuthResult.Failure` if not configured.
- `AdminRepository` checks `nexus_admins/{uid}` in Firestore; returns `false` on any error (offline-safe).
- `UserSession.Authenticated.isAdmin: Boolean` propagates through `SessionManager.setAuthenticatedSession(isAdmin=)`.
- `StubFirebaseAuthRepository` still exists as fallback class in `AuthRepository.kt` but is NOT wired in DI.

**Why:** Graceful degradation — app fully works without real Firebase (guests, offline), but auth + sync activate when credentials are present.

## Health Vault Security Contract
- Text records → `HealthVaultRepository`: DataStore (local, always) + Firestore (only text fields, only when Google-authenticated).
- **Cards, documents, passwords → NEVER synced; stay on device with BiometricVault AES-256-GCM.**
- Biometric gate in `NexusHealthVaultScreen`: `isLocked = true` on entry, `LaunchedEffect(Unit)` triggers `BiometricPrompt`, auto-unlocks if no biometric enrolled.

## Admin Notification Gate
- `NotificationRepository.sendAdminNotification(title, body, category, session)` checks `session.isAdmin` before writing.
- `NotificationCenterScreen` shows `ExtendedFloatingActionButton` + `AdminNotifDialog` only when `session.isAdmin == true`.

## New Features (v1.2)
- `TotpAuthenticatorScreen`: Pure-Kotlin TOTP (HMAC-SHA1 + Base32 decode), no external library. Accounts stored in local `remember` state (can be persisted to DataStore in future).
- `NetworkSpeedTestScreen`: OkHttp ping/download/upload, animated arc gauge, single LaunchedEffect-triggered button (runCount pattern).
- `MergePdfTool`: Real implementation using `PdfRenderer` (read each page) → `Bitmap` → `PdfDocument.Page` (write). Android-only, no third-party lib needed.

## How to Apply
- Any new feature using Firestore must only sync small text fields — never binary data or sensitive credentials.
- Admin features must always check `session.isAdmin` before writing — never trust client-side alone; add Firestore Security Rules server-side.
- New screens follow: FeatureId → Screen → FeatureCatalog → NexusNavHost → AppModule (ViewModel if needed) → index.html.
