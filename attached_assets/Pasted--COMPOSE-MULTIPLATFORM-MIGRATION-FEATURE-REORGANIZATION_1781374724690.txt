# COMPOSE MULTIPLATFORM MIGRATION & FEATURE REORGANIZATION

## Complete Compose Multiplatform Audit

Perform a deep scan of the entire project and identify all code that is not properly organized for Compose Multiplatform.

Current project structure appears fragmented, inconsistent, and difficult to maintain.

Reorganize the entire codebase into a proper Compose Multiplatform architecture.

Target platforms:

* Android
* iOS
* Desktop

Maximize code sharing.

Target:

* 90%+ commonMain code
* Minimal platform-specific code

---

# COMPLETE FOLDER REORGANIZATION

The current folder structure is confusing and difficult to navigate.

Rebuild the entire folder structure.

Requirements:

* Consistent naming
* Consistent architecture
* Minimal nesting
* Easy navigation
* Predictable file locations

Every developer should be able to find any screen, ViewModel, repository, or service within seconds.

---

# FEATURE REBUILD REQUIREMENT

Most features currently consist of a single screen that contains too much functionality.

This creates poor usability and poor maintainability.

Completely rebuild all major features into multiple modules and multiple screens.

Do not keep large all-in-one screens.

---

# COMMON FEATURE FRAMEWORK

Every feature must use the same structure.

feature/

presentation/

screens/
components/
state/
actions/

domain/

models/
usecases/

data/

repository/
datasource/

navigation/

No feature should use a custom architecture.

All features must follow the same pattern.

---

# FAVORITES EXPERIENCE REDESIGN

Do NOT add a visible Favorite button to every feature.

Avoid UI clutter.

Instead implement:

### Add to Favorites

* Double Tap and Hold
* Long Press
* Context Menu Action
* Accessibility Action

### Remove from Favorites

* Double Tap and Hold
* Context Menu
* Accessibility Action

---

# ACCESSIBILITY FAVORITES SUPPORT

TalkBack users must be able to manage favorites.

Add semantic actions:

* Add To Favorites
* Remove From Favorites
* Pin To Home
* Unpin From Home

These actions must be available through:

* TalkBack Actions
* VoiceOver Actions
* Keyboard Actions
* Context Menus

Users must not be forced to use gestures.

---

# UNIVERSAL FEATURE ACTION SYSTEM

Every feature card should support:

* Open
* Pin To Home
* Add To Favorites
* Share
* View Information

through a unified action system.

No duplicate implementations.

---

# HOME SCREEN IMPROVEMENTS

Home screen should automatically display:

* Favorites
* Recently Used
* Most Used
* Pinned Features
* Suggested Features
* New Features
* Notifications

Users should not need to manually configure the dashboard.

---

# SEARCH & DISCOVERY IMPROVEMENTS

Everything must be discoverable.

Search should find:

* Features
* Screens
* Tools
* Documents
* Images
* Settings
* Actions
* Favorites

Create a universal search engine shared across all modules.

---

# FINAL GOAL

Transform the project from a collection of large single-screen features into a fully modular Compose Multiplatform application with:

* Clean folder structure
* Shared architecture
* Shared components
* Multiple modules per feature
* Better accessibility
* Better discoverability
* Better maintainability
* Better cross-platform support
* Minimal UI clutter
* Consistent user experience

Every feature should feel like part of the same ecosystem rather than a separate application.
